SPA i18n

SPA JS has built-in support for internationalization ( i18n ). The i18n labels can be used/managed in template as well as in script of any component within SPA JS.

Language Code

In SPA JS, itâ€™s recommended to use the ISO Language Code along with the ISO Country Code separated by underscore ( _ )

Example: en_US, ja_JP, es_ES

For List of ISO language code refer: https://www.w3schools.com/tags/ref_language_codes.asp

For List of ISO Country code refer: https://www.w3schools.com/tags/ref_country_codes.asp

Language Files and Folder

Create a folder "language" in app folder to store the language files. The language file name must follow the naming convention as Language_lc_CC.txt where lc is ISO Language code in lowercase and CC is ISO Country code in Uppercase.

NOTE: The Folder Name and File Names are case-sensitive.

i18n Language File Content

The i18n Language file contains a key value pair. These keys will be referred in the Component's template or script.

Enable i18n

To enable i18n feature, in the HTML body tag add attribute i18n-lang with value of default i18n Language Code ( lc_CC )

Change Language

The default i18n language can be changed using HTML elements (on click) or JavaScript.

Using UI element

Add attribute data-i18n-lang with the i18n Language code. This will register onClick events for those elements to trigger Language Change.

Using Script

Use spa.i18n.setLang method to change the language using script manually.

onLangChange

Optionally define your custom function spa.i18n.onLangChange to call before the language change. The Optional return value of this function overrides the language that is about to be set.
