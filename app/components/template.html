Component's Base Properties

template

Specify the template string (html tags). Use Handlebars syntax for data-binding. Refer Component's UI section for more details.

SPA JS highly recommends to keep HTML in .html file and Javascript in .js file. Use this 'template' property for a few lines template only. For the larger template use componentX.html
NOTE: Starting v2.84.0+, the component structure has been simplified to define its properties using simple independent variables prefixed with underscore ( _ ).
Prior to v2.84.0: A function spa.$( 'componentX' , { } ) is used to define Component's base properties (without underscore ( _ ) prefix like template, target, dataUrl etc.) and extended properties like events and other custom actions to be performed on component UI.
Single line template: Multi-line template: use back-tick ` not single-quote!
template is an optional property. If not specified, SPA JS will load the corresponding template from componentX.html file, when the component is rendered using $.render method or data-spa-component attribute.
All base properties can be overridden before the render call using spa.$render( ) method.
