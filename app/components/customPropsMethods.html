Component's Extended Properties

Default Properties/Methods

app.componentX.$data the data, which has been used in template.

app.componentX.onRender function, which will be called just before rendering (spa.$render). (from v2.54.1)

app.componentX.renderCallback function, which will be called after rendering (spa.$render).

app.componentX.onRefresh function, which will be called just before rendering (spa.$refresh). (from v2.54.1)

app.componentX.refreshCallback function, which will be called after refresh (spa.$refresh).

app.componentX.onRemove function, which will be called just before getting removed from the DOM. can be stopped by return false; (from v2.54.1)

app.componentX.events the collection of events, which will be registered on component's DOM elements after render or refresh.

Custom Properties/Methods

A component can have its custom properties and methods

Accessing Component’s Extended Properties and Methods

> in Script (componentX.js)

The component’s extended properties and methods can be accessed using app.componentX.PropertyOrMethodName or app.componentX[ 'PropertyOrMethodName' ]

Within extended methods, the properties or methods can be accessed using this. keyword.

Note: this.$data refers to the Component’s Final Data (base property) i.e. The final data which goes to the template. (Refer above example)

> in HTML (template) (componentX.html)

The component’s extended properties can be accessed using {{ {@$}propertyName }} for the current component's propertyName or {{ {@$componentY}propertyName }} for componentY's propertyName.

The component’s extended methods can be accessed using onClick="{$}methodName();" for the current component's methodName or onClick="{$componentY}methodName();" for componentY's methodName.

Important note on component's $data property:

The $data is available by default to all the components. However, if required, it can be disabled/enabled using global or component's property.
