SPA API Mock

SPA JS comes with built-in mocking system for faster development.

Enable API Mocking

To enable mock for all the API URLs defined in spa.api.urls, set spa.api.mock to true (by default mock system is disabled).

API Root Context (for mock data)

If you require SPA’s mocking (static) data for the API URLs, It’s highly recommended to have the single API root context and the same to be specified in app.api.liveApiPrefix

This Live API Root context is must for SPA Mock System to be enabled.

Mock Data (static) Structure

Create data.json file in MOCK API URL folder structure with mock response data.

If you are not getting the mock data, watch browser's developer tool and check for 404 error to find the actual path for mock data.

Mock Data (static) Folder Root (api_)

The default mock (static) folder root (api_) can be changed using app.api.mockRootFolder key.

The path is with reference to the start page (e.g. index.html). If the mock data is hosted on a remote server, the full root path should be provided (e.g. https://mock.server/api).
